# Make programs.yaml Path Configurable

## 概要
現在、`programs.yaml`のファイルパスは固定されていますが、CLIからファイルパスを指定可能に変更します。

## 現状の問題点
- `programs.yaml`のファイルパスが固定されており、柔軟性がありません
- 複数のデータベースファイルを管理する際に不便です

## 解決策
- CLIに`--data-file`オプションを追加し、任意のYAMLファイルをデータベースとして使用できるようにします
- 既存の`config.py`の`programs_file`パラメータと連携し、CLIオプションが優先されるようにします

## 期待される影響
- 既存のユーザーには後方互換性が維持されます
- 新しいオプションにより、柔軟なデータ管理が可能になります
